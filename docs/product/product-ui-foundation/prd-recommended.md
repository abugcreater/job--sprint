# 推荐方案：单任务作战台

日期：2026-07-10

## Visual thesis

克制、可信、能执行的求职决策工作台：深墨色建立结构，低饱和青绿只标主动作，暖中性色承载长时间使用，细线和排版取代卡片瀑布。

## Content plan

### 全局产品壳

1. 品牌与产品定位。
2. 按用户任务分组的主导航。
3. 当前页与账号状态。
4. 同步状态与真实恢复入口。

### 今日工作台

1. 日期、当前阶段、唯一主动作。
2. 当前任务、产出与完成条件。
3. Evidence Gate 与保存反馈。
4. 风险、下一任务、口述、资料等次级上下文。

新用户没有画像时，不展示任务和 KPI；改为一条三步建档路径，第一步是唯一可执行动作。

## Interaction plan

- 页面主要区块使用一次性轻微上移进入；`prefers-reduced-motion` 时关闭。
- 导航选中态从大色块改为细条与稳定底色，减少页面跳动。
- 同步状态不再提供没有真实实现的“重试成功”反馈，统一导航到“更多”中的同步与恢复区。
- 移动端按“当前状态 -> 主动作 -> 下一步”重排，不把桌面两栏机械压缩。

## Style Library Reference

- 主风格：`swissDesign + flatDesign`
- 辅风格：`utilityFirst`，只用于密度、状态与表单纪律。
- 拒绝：`glassmorphism`、`gradients`、营销 hero、暗色数据室、卡片瀑布、圆形匹配分、霓虹图表。
- 产品匹配：求职执行需要承认信息缺口和下一步，不需要机会感包装。

## Component Recipe

- 复用：路由导航、状态 chip、按钮、字段控件、Evidence Gate、现有 store。
- 重构：产品壳、页面标题带、进度状态带、当前任务工作面、新用户步骤脊柱。
- 允许新增：上下文 rail、真实同步导航、移动端安全区适配类。
- 避免：无业务对象的统计卡、重复白卡、装饰性图表、伪交互按钮。

## Density and whitespace

- 桌面：中高密度；1440px 主工作区占据可用宽度，右 rail 只承载决策上下文。
- 移动：中密度；390px 首屏必须看到当前阶段、一个主动作和下一步，不让同步卡重复占屏。
- 宽度：保留可读行长，但取消所有页面统一窄居中的倾向。
- 空白：用于分组，不用于掩盖缺少内容层级。

## 状态矩阵

| 状态 | 必须呈现 | 恢复动作 |
|---|---|---|
| 无画像 | 三步建档路径、数据边界 | 导入简历或 JD |
| 有画像无日历 | 当前画像、未完成步骤 | 生成今日日历 |
| 有当前任务无证据 | 主任务、完成条件、证据缺口 | 补证据 |
| 有证据 | 可标记完成、读回摘要 | 标记完成或继续补充 |
| 本地模式/同步失败 | 当前数据仍可本地记录 | 前往同步设置 |
| 保存成功 | 文本回执 | 继续下一动作 |
| 输入不完整 | 文字校验，不只靠颜色 | 补齐字段 |

## Web 与 Android 验收

- 1440x1100：首屏主任务明确，宽屏没有无意义大白边，不出现 KPI 卡阵列。
- 390x844：主动作在首屏可见，底部导航不遮挡内容，触控目标至少 44px。
- Android WebView：使用同步后的 React assets；状态栏、底部安全区、键盘输入和本地存储流不回退。
- 无障碍：语义区域、可见焦点、`aria-current`、状态回执和 reduced motion 有效。
