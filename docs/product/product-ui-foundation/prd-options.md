# UI 基线构图方案

日期：2026-07-10

## 方案 A：数据仪表盘

首屏以 4 到 6 个 KPI 卡、趋势和模块入口为主。

- 优点：信息覆盖多，开发成本较低。
- 缺点：用户每天的核心任务被统计淹没；继续强化通用 SaaS 模板和卡片阵列。
- 裁决：不选。

## 方案 B：单任务作战台

首屏围绕“现在做什么、为什么做、完成条件是什么”组织。当前任务和证据门是主工作区，风险、下一任务与资料进入次级上下文 rail。

- 优点：与 Job Sprint 的每日执行模型一致；桌面和 Android 移动端都能保持同一任务顺序。
- 缺点：需要重排现有组件层级，不能只换颜色。
- 裁决：采用。

## 方案 C：求职旅程时间线

首屏以画像、知识、机会、面试、复盘的阶段时间线为主。

- 优点：产品全貌清楚，适合阶段复盘。
- 缺点：今日行动被稀释；用户进入后仍需二次寻找当前任务。
- 裁决：延期到复盘或统计模块评估。

## 裁决依据

当前基线截图证明：页面并非缺少功能，而是缺少明确主次；宽屏内容被多个同权卡片切碎，移动端同步状态与建档引导重复占据首屏。方案 B 能直接消除这两类问题。
