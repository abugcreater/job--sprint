# AI 运行恢复提示推荐方案

日期：2026-07-24

## 用户结果

用户每次生成 AI 草稿后，都能知道“草稿是否可用、真实模型是否参与、现在应该自己做什么”，而不是把所有 fallback 都理解成模型失败。

## 最小实现

1. React runtime client 对认证失败、服务端不可用和非预期响应产生有限、可公开的诊断码。
2. Coach 生成流程将诊断码写入已有 `llmRuns.warning`，同时提供对应反馈文案。
3. `diagnoseLlmRun` 为已知诊断码提供统一标签、解释和恢复动作。
4. Coach 运行记录与 Stats 最新诊断复用同一映射。

## 约束

- 不向浏览器暴露 token、URL、堆栈、原始 provider body 或服务器环境变量。
- fallback 草稿仍是可审核的草稿，不能因诊断 UI 改动而自动写入正式日程或知识边界。
- 对未知错误只展示安全的通用恢复提示，保留服务端日志作为维护路径。
