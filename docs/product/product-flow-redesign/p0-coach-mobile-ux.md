# P0：准备工作台与 Android 可达性

## 验收目标

- Coach 不再同时渲染所有工作区。
- 移动端一次只显示一个准备阶段，默认阶段由真实数据决定。
- 每个阶段可在合理手势次数内到达最后操作。
- 锚点标题不会被移动 sticky header 遮挡。
- IME 打开时底栏隐藏，当前字段、下一字段和保存动作可达。
- Knowledge、Applications、Interview、Review 在移动端有明确入口。

## 阶段定义

| 阶段 | 完成条件 | 主内容 | 下一步 |
|---|---|---|---|
| 画像 | 存在有效画像 | 导入建档或详细画像表单 | 确认知识边界 |
| 知识边界 | 当前画像至少 1 条边界 | 边界提取与正式边界 | 生成今日计划 |
| 今日计划 | 当前日期至少 1 条日程 | 日程表单与精简列表 | 处理 AI 建议 |
| AI 建议 | 至少 1 条建议已处理 | 生成、编辑、接受或拒绝 | 回到今日执行 |

## Android 验收矩阵

| 场景 | 通过条件 |
|---|---|
| 无画像冷启动 | 当前阶段和主 CTA 首屏可见 |
| 阶段切换 | 只出现当前阶段的表单和记录 |
| 画像表单 + IME | 底栏隐藏；保存按钮可通过滚动到达 |
| 边界 textarea + IME | 输入末尾和操作按钮可达 |
| 手势导航 | 底栏不压系统手势条 |
| 三键导航 | 无双重底部留白 |
| 物理上滑 | 最后操作顶部高于底栏 |
| 字体比例 | 1.0 与 1.3 均无横向溢出 |
| 持久化 | 保存后杀进程重启读回 |

## 非目标

- 本阶段不新增画像字段。
- 不改 AI 生成合同。
- 不把所有历史记录一次展示在移动端。
- Native inset 仅在真机证据证明仍有错误时修改。
