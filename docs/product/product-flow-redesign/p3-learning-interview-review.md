# P3：学习、面试与复盘闭环

日期：2026-07-10

结论：`PASS_WITH_LIMITS`

## 目标

把三个已有功能页从连续卡片堆叠改为一条可执行旅程：

`学习笔记 → 同任务单题练习 → 口述证据 → 复盘事实/卡点 → 明日第一步`

## 采用的构图

### 学习工作台

- 默认只显示当前任务和学习笔记入口。
- “当前任务 / 任务知识摘要”互斥切换，知识摘要不再堆在任务页面下方。
- 资料标签和最近笔记放入按需展开区；移除最近笔记的嵌套滚动。
- 保存笔记后提供唯一出口“用这条笔记练一道题”。

### 面试单题会话

- 移动端首先显示当前题和回答框，任务、历史与题库都降为次级展开区。
- 保存口述后提供“去复盘这次练习”。
- 原“AI 评分”改为“本地规则自检”；0–100 内部值不作为主视觉。
- 新证据写入“规则覆盖”，仍兼容读取历史 `AI评分` 字段。

### 今日闭环复盘

- 默认先写复盘，再看整理；“写复盘 / 看整理 / 历史”互斥切换。
- 首屏只要求可证明结果、最大卡点和明日第一步。
- 面试表达、知识边界和薄弱回答进入可选折叠区。
- 复盘删除必须二次确认。
- “AI 分析”改为“本地规则整理”；任意权重生成的“闭环分”从界面移除，周复盘只显示证据、覆盖和完成事实。

## 当前限制

- 本地 store 写入为同步接口，尚无可注入的持久化失败合同；不制造假 loading。
- 面试规则仍使用内部数值计算以保持历史合同，但界面只展示覆盖级别、命中项、缺口和追问。
- 复盘历史编辑尚未统一 URL 状态；后续与全局草稿恢复协议一起处理。
